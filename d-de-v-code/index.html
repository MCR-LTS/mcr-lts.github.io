近日，编程领域忽然传来了一个轰动业界的消息——
某 20 岁天才少年无师自通，在学校 Java 课程开课之时，便写完了一整个 Java 程序。
顿时，全世界的目光都聚焦在了他的身上我就是全世界。

这位少年解决的，正是近年来，中国网民登录网站的最大痛点：
明明是在电脑端登录，却要先在手机上查看短信验证码，再在电脑上逐字输入；
倘若你前一天晚上*多了，一眼记不住的话还得看好几眼，严重影响办事效率。
带着拯救世界的信念，他编写了一个将这段过程化繁为简的程序——
手机收到验证码后，会自动同步到电脑剪贴板中，直接 Ctrl+V 就能无痛完成填写。

既可涵养心境，又能建立心流，此乃改变互联网浏览体验之神器也。
但非常遗憾的是，这位少年没有发布程序，也没有开源代码。
没办法，只能尝试突破技术封锁，自己探索一个了。

居然硬夸这么多了

其实这就是当下最流行的 vibe coding，他是这么说的，我也是这么做的，所以标签是随便乱写。
AI 时代是这样的，有想法就能变成现实，包括领导降本增效的想法

基本指令

项目初始提交时，在本地目录依次执行以下指令：

git init
把文件夹变成 Git 仓库——创建一个隐藏的 .git 目录，记录版本历史，跟踪文件变化。什么政府宣传公文

git add .
把当前目录下 (.) 的所有改动加入“暂存区”。未来如需提交文件变动，将 . 换成被修改的文件路径。建议搭配 .gitignore，避免提交缓存、日志尤其是 .DS_Store 等无关文件。

git commit -m "First Commit"
把暂存区内容正式做成一次提交，-m 后是提交说明。作为项目历史中的一个快照，每次提交都可便捷查看与还原。

git branch -M main
把当前分支重命名为 main。-M 是强制版（即使有同名分支也覆盖），目的是和 GitHub 默认主分支名保持一致（本地分支名有时会是 master）。

git remote add origin [Github项目链接].git
把本地仓库与远程地址绑定，一次绑定终身使用。如果这一步或下一步提示输入账号密码，大概率是初始配置错了。

git push -u origin main
正式把代码/修改推送到 GitHub。origin 是刚才设置的远程名，main 是分支名，-u 用来建立“上游追踪关系”。
如果创建存储库时加入了任何文件，将会提交失败，此时可在参数后添加 -f。

其中，第 2/3/6 步每次修改都要执行，前两步视情况修改参数，而最后一步去掉所有参数直接 git push 即可。

除这些外，常用的命令还有：

  • git status 看当前状态
  • git pull --rebase 拉最新代码避免冲突
  • git diff 看工作区和暂存区差异
  • git diff --staged 看已经 add 但还没 commit 的内容
  • git log --oneline --graph --decorate --all 看分支分叉和合并
  • git branch 看本地分支
  • git switch -c 新分支名 新建并切换分支
  • git switch 分支名 切换已有分支
  • git merge 分支名 把某分支合进当前分支
  • git remote -v 看远程地址
  • git fetch 只下载远程更新但不改当前文件
  • git pull = fetch + merge
  • git restore 文件名 撤销工作区改动
  • git restore --staged 文件名 add 后撤出暂存区
  • git commit --amend 修正最后一次提交
  • git reset 往前退版本
  • git revert 提交ID 生成反向提交
  • git stash 暂存现场
  • git stash pop 恢复

「客户端通过点击一个按钮，自动获取服务端 ip 地址并填写」
幸运的是，这个方案是可以落地的：

  • 手机点击“自动发现电脑”
  • 手机往 255.255.255.255:9999 发一条 UDP 消息，比如 DISCOVER_SMSSYNC
  • 电脑端监听 9999
  • 电脑收到后，回复 SMSSYNC_SERVER: 192.168.1.23:8888
  • 手机收到回复，自动把 192.168.1.23 填到输入框，并保存

语法解析

写都写（并非 出来了，还不顺便了解一下 Java 的基础语法？
下面的笔记大多确实是复制粘贴，但在润色文章的过程中，我也在持续吸收、理解与思考这些概念
——因为表达能力薄弱，我在润色上下的工夫远比正常人多数倍。

类与导包 —— 程序的组织形式

在 Java，代码不能脱离“类”独立存在，类是组织代码的基本单元。

import：Java 自带了庞大的标准库，当代码需要用到这些现成的功能时，必须先在文件开头用 import 声明，告诉编译器去哪里找这些依赖。跟 python 差不多。

public class ClipboardServer{}：

  • class：声明这是一个“类”——一组数据和行为的集合体。
  • public：访问修饰符，表示这个类是公共的，对外部可见。
  • ClipboardServer：类名。公共类的类名必须与文件名完全一致。

程序的入口

计算机执行任何代码都需要一个明确的起点。
public static void main(String[] args) 是 Java 程序的标准入口方法。Java 虚拟机运行程序时，会主动寻找这个方法，并顺序执行里面的代码。应该类似于 int main？
如果没有这个方法，程序可以编译，但无法运行。

数据类型与变量

程序运行的本质就是处理数据，Java 必须明确声明每个数据的类型（强类型特性）。
基本数据类型如 int、boolean，引用数据类型如 String。

message 是变量，它的值可以在运行中改变。
PORT 前面加了 final 关键字，表示它是常量，赋值后，内存中的这个值就被锁定了，后续代码不得试图修改。

方法/函数 —— 封装逻辑代码

为了代码复用和结构清晰，把特定的功能封装成一个个独立的代码块，称为“方法”。

一个标准的方法声明包含：

  • 访问控制修饰符：这里是 private，表示这个方法只能在 ClipboardServer 这个类的内部被调用，外部不可见。
  • static：表示这个方法属于这个类本身，不需要通过 new 实例化对象就可以直接调用。
  • 返回 void 表示这个方法只执行过程，执行完毕后不返回任何数据给调用者（例如 setClipboard 只是负责复制）。
  • 返回 boolean 表示这个方法执行完毕后，必须用 return 向调用者返回布尔值结果（例如 isMac 返回是不是 macOS）。
  • 参数列表：括号里的 (String text)，表示调用时必须强制传入一个字符串类型的数据。

控制流 —— 逻辑分支与循环 （几乎与 C 语言相同）

代码默认是顺序执行的，控制流语句用于改变执行顺序。
条件分支 (if-else)：根据括号内的布尔值决定执行哪段代码。
循环控制 (while)：括号内的条件为真时持续运行，条件写死为 true 的话就构成了无限循环。在服务端编程中，服务器必须一直挂起运行，等待客户端接入，而不能执行一次就退出。

面向对象与实例化

new 关键字：在 Java 中类只是模板，要想真正在内存中使用某个复杂类型的功能，需要用 new 在内存中实例化一个具体的对象。
比如，根据 StringSelection 这个类模板，在内存中创建一个真实存在的对象，并将其内存地址赋值给变量 selection。

异常处理 —— 保证程序健壮性

代码中涉及到网络 I/O 和系统进程操作，极易受外部环境影响：

Java 强制要求对可能出错的外部交互代码进行处理。
try 中放正常逻辑，catch 块捕获特定类型的异常。如果 try 中的代码发生错误，程序会将控制权立即转移到 catch 块中，使得开发者有机会打印错误日志或进行降级处理。
