Discord 一起听歌 Bot

Mercurylotus04/02/26

大家早上中午下午晚上好呀。欢迎收看这篇或许是本博客迄今为止最短的文章（

故事很简单，我想在语音频道一起听歌，便想 vc 一个 bot。
原因很简单，找不到好用的，且没有国内音乐源。
想法很简单，我想让机器人接入网易云，破解会员歌曲，输入歌单链接自动随机播放。
过程……我希望很简单。

那我们开始吧。

框架

Bot 一定要注册（恩情
打开开发者门户，选择注册机器人，进入 dashboard。
点击右上角的 “新 APP”，给你的宝宝起名，为了防止是机器人繁衍后代，还需要进行人机验证。
进入信息页面，左侧 机器人 -> 令牌 -> 重置令牌，复制他，不要在任何场合公开。
最后勾选同一个页面下方的“消息内容意图”和“服务器成员意图”，保存更改后就完成了基础配置。

左侧 OAuth2，先复制客户端 ID 备用，再在下方“URL 生成器”选择“applications.commands”和“bot”。
在接下来的弹窗中，勾选发送消息，连接和讲话，复制最下方的 URL 到浏览器中，选择要加入的服务器，完成万里长征第一步。

是故意的还是不小心的

依赖

先安装 FFmpeg 并配置好环境变量。Discord 语音频道使用特定的音频流，播放流媒体平台的音乐需要依赖 FFmpeg 进行实时转码和推流。
然后准备好 Node.js，使用 discord.js 库开发 Bot 是比较适合新手的方案。
最后找个网易云的 API。众所周知，NCMApi 已经纪了两年了虽然还有 30000 个 fork 可以拿来用，但我还是搜索了下有没有比较新的更好的 API。
然后我发现 NCMApi 还没纪

付费破解怎么办？有一个方案是登录一个有 VIP 的账号，但是你也懂这方案风险有多高。
所以参考 UnblockNeteaseMusic 的思路。机器人只提取歌名 + 歌手，去其他平台搜索这首歌，提取音频进行播放。

上线

进入项目文件夹，初始化并安装必要的库。

配置环境变量与代理。token 填写刚才获取的，代理看软件， clash 是 127.0.0.1:7890，v2ray 通常是 :10809。

懒筒

上篇提到，我这次要全程使用 lazygit。

首先建立 .gitignore，把 node_modules/ 和 .env 放进去。
终端 lazygit，先按 y，再直接回车跳过，进入 GUI。
按 a 暂存所有文件，按 c 提交，输入提交信息后按下回车。
下一步…好家伙绑定仓库这一步还要用原始方式输入命令…
按 : 调出命令行，输入最不 lazy 的 git remote set-url origin git@github.com:username/dcbot.git ，如果有什么多余的设置就凭感觉填写。
最后 shift+p 推送。如果问你问题就喊是。

初试

node index.js 运行脚本，shell 应显示

同时 Discord 中的 bot 也应显示在线。
加入任何一个语音频道，再在任何一个文字频道输入 !join，机器人就会进入你的频道。

但是要退出的话就只能管理员强行踢了
